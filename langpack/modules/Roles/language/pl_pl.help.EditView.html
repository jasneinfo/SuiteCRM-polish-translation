Role

Use this page to create roles and grant them access to Sugar modules.

To create a role, enter the following information and click Save:

  • Name: Enter a name for the role.
  • Description: Enter a brief description of the role.

A list of current permissions for existing modules display on the page. Specify access to one or more modules. By default, access is enabled for all modules.

  • To deny access to a module, click Not Set corresponding to the module in the Access column, and from the drop-down list, select Disabled.
  • To specify end-user privileges or administrator privileges, click Not Set in the Access Type column and select Normal or Admin respectively.
  • To allow a specific privilege for a module, such as editing or deleting records, click Not Setin the appropriate Action column, such as List for ListView or View for DetailView, corresponding to the module and select one of the following:
    • All. Allows all users of the specified Access type to delete a record in the module.
    • Owner. Allows only the record owner to delete the record.
    • None. Prevents all users of the specified Access type from deleting records in the module.

    To create the role, click Save; click Cancel to exit the page without saving your changes.

    To duplicate the set of permissions, on the Detail page, click Duplicate, and enter a name for the new role.
